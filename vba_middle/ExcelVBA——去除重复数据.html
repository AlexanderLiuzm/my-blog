# 前期工作

先利用录制宏来获取最基础的去除重复项的语句

ActiveSheet.Range("$A$1:$F$968").RemoveDuplicates Columns:=Array(1, 2, 3, 4, 5, 6) _, Header:=xlYes

获取到的这个语句块中的对象还是静态的，并不能自动适应任意一个表格，为任意一个表格进行删除重复项，因此我们要对上述的代码块进行改造。

# 确定要改造的对象

先观察这这行代码，发现其中的对象有两个——

  1. range-Range("$A$1:$F$968")
  2. Array-Columns:=Array(1, 2, 3, 4, 5, 6) _

所以我们就想办法改造这两个对象，以达到让宏代码自己适应不同的数据表并且进行重复项删除。

下面是经过改造后得到的宏代码，'后面的是注释

Sub 宏1()
'
' 宏1 宏
'

Dim arr()
'声明i
Dim i As Integer
'声明数列的总量
ReDim arr(0 To ActiveSheet.UsedRange.Columns.Count - 1)
'从第一行到最后一行开始循环，自动检测并建立一个包含所有数据列的数组
For i = 0 To ActiveSheet.UsedRange.Columns.Count - 1
    arr(i) = i + 1
Next i
'删除重复项的语句
'对象是有数据的单元格
'对这些单元格用RemoveDuplicates方法
'后面的对象是列
    Range(ActiveSheet.UsedRange.Address).RemoveDuplicates arr, Header:=xlYes
End Sub

# 对Range对象的改造

首先需要明白的是Range这个对象主要用于确定执行查找重复项的具体范围，即要进行删除重复项的那个表格是从第几行第几列到第几行第几列。所以我们可以用Usedrange作为一个对象，在一个表格中所有含有数据的单元格我们都认为是要进行校验的对象。又因为进行方法的对象是一个数据表的具体区域，所以应该用Address对象。

Range(ActiveSheet.UsedRange.Address)

# 对Array对象的改造

这段代码中的Array对象主要是确认这个表格有几列，即有多少个字段。我们要创立一个新的数组，利用一个循环来对这个数组进行定义，让它变成一个动态数组。

For i = 0 To ActiveSheet.UsedRange.Columns.Count - 1
    arr(i) = i + 1
Next i

如果想让i从1开始，写成下面这样行不行呢？

For i = 1 To ActiveSheet.UsedRange.Columns.Count
    arr(i) = i
Next i

经过我的实验，，是不行的，大概是因为VBA默认是要从0开始的，反正记住这个规则就好了。

# 改造完成的代码

经过上述的两个步骤完成的代码是这样子的

Dim arr()

Dim i As Integer

ReDim arr(0 To ActiveSheet.UsedRange.Columns.Count - 1)

For i = 0 To ActiveSheet.UsedRange.Columns.Count - 1
    arr(i) = i + 1
Next i

ActiveSheet.Range(ActiveSheet.UsedRange.Address).RemoveDuplicates Columns:=arr, Header:=xlYes

但是如果真的尝试在VBA中运行这段代码，Excel会提示代码有错误存在。这是为什么呢？

第一个错误是Range前面有ActiveSheet明显属于重复，后面的括号中已经规定了对象的范围是ActiveSheet中的Usedrange，前面就不用再定义ActiveSheet，删除即可。

第二个错误是arr前面有Columns:=。arr的定义中包含ActiveSheet.UsedRange.Columns，所以前面的Columns:=也是属于多余的。当然这两个错误的原理是我的猜测，具体为什么我也不甚清楚，我就用我自己猜测的两个原理来辅助记忆这个语法规则。
